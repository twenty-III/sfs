Live endpoints — click to try

Each button fires a real request to this server and shows the raw response.

GET /ping Health check

Returns pong. Used by load balancers to verify the server is alive.

GET /info Server metadata

Returns live JSON with server name and the number of active worker threads.

GET /echo?msg=… Query parameters

Demonstrates query string parsing. The server reads req.query_params() and echoes msg back.

GET /users/:id Path parameters

The :id segment is captured at route-match time and placed in req.path_params().

POST /users Request body

Demonstrates body reading. The parser uses Content-Length to know how many bytes to read from the socket.

DELETE /users/:id 204 No Content

Returns 204 — success with no response body. The correct HTTP status for a successful delete.

What's inside

Thread Pool

Fixed std::thread workers share a queue protected by std::mutex and std::condition_variable. Workers sleep for free when idle. Spawning a thread per request costs ~10–50µs; pool workers pick up tasks in microseconds.

thread_pool.hpp / .cpp

Request Parser

Reads raw bytes from the socket in a loop — TCP delivers data in chunks, not messages. Parses the request line, headers, query string, and body. Rejects oversized headers and truncated bodies.

request.hpp / .cpp

Router

Stores (method, pattern, handler) triples. Matches paths segment-by-segment, capturing :param wildcards. Returns 405 when the path exists but the method doesn't — correct HTTP semantics.

router.hpp / .cpp

Response Builder

Content-Length and Connection are auto-managed — set_header() rejects them to prevent duplicate headers. status_text_for() returns std::string_view into literals: zero allocation per call.

response.hpp / .cpp

File Server

Serves HTML, CSS, JS, images from disk. Falls back from the router on 404. Blocks path traversal (../ attacks) by scanning segments. MIME type from extension so the browser renders, not downloads.

file_server.hpp / .cpp

Logger

Meyers singleton, thread-safe via std::mutex. Variadic template with a C++17 fold expression lets you pass any number of typed values. __FILE__ and __LINE__ captured at call site via macros.

logger.hpp / .cpp

curl cheat sheet

# Health check
curl http://localhost:8080/ping

# JSON metadata
curl http://localhost:8080/info

# Query parameters
curl "http://localhost:8080/echo?msg=hello"

# Path parameters
curl http://localhost:8080/users/42

# POST with a JSON body
curl -X POST http://localhost:8080/users \
     -H "Content-Type: application/json" \
     -d '{"name": "Alice"}'

# DELETE — expect 204 No Content
curl -s -o /dev/null -w "%{http_code}" \
     -X DELETE http://localhost:8080/users/42

# See full response headers
curl -i http://localhost:8080/ping
